# デザインレビュー依頼：クイックオーダー商品情報レイアウトの縦サイズ削減

## 依頼日
2025-11-07

## 依頼内容
クイックオーダーの商品情報表示部分が縦に大きく、ユーザビリティを低下させているため、**縦サイズを削減したコンパクトなレイアウト**への変更を検討したい。

## 現状の課題

### 1. 商品情報表示が縦に大きい
- **廃番商品の警告ボックス**: 画像64x64px + 複数行のテキスト情報（商品名、コード、廃番日、理由）
- **代替商品リスト**: 縦並びで各商品が大きなスペースを占有（画像48x48px + 商品情報 + ボタン）
- **通常商品の表示**: 画像64x64px + 商品名、価格、在庫情報

### 2. パディング・余白が大きい
- ボックス内パディング: `p-3`（12px）
- 要素間の間隔: `gap-3`（12px）
- これにより全体的に縦サイズが増大

### 3. 1画面に表示できる行数が少ない
- 現在: 約3-4行程度しか見えない
- 期待: 5-6行以上を1画面で確認できるようにしたい

## 改善目標
- **廃番商品表示**: 縦サイズを40-50%削減
- **通常商品表示**: 縦サイズを20-30%削減
- **視認性**: コンパクトにしつつも、必要な情報は明確に伝える
- **可読性**: テキストサイズは可読性を損なわない範囲で縮小

## デザイン検討ポイント

### A. 画像サイズの調整
- 現在: 廃番商品64x64px、代替商品48x48px、通常商品64x64px
- 提案: すべて統一して48x48pxまたは40x40pxに縮小
- **質問**: 商品画像は何pxまで縮小可能か？（視認性とのバランス）

### B. 廃番商品情報のコンパクト化
現在の表示項目：
- 警告アイコン + 「この商品は廃番になりました」
- 商品名（1行、line-clamp-1）
- 商品コード
- 廃番日（例: 2024-12-01）
- 廃番理由（例: モデルチェンジのため廃番）

提案1: 情報を統合
```
⚠️ この商品は廃番になりました（2024-12-01 | モデルチェンジのため）
コード: DM110BK
```

提案2: 商品名を削除（コードのみ）
```
⚠️ 廃番商品（コード: DM110BK）
2024-12-01 廃番 | モデルチェンジのため
```

**質問**:
- 廃番商品の商品名は必要か？（コードだけでユーザーは理解できるか）
- 廃番日と理由を1行に統合してよいか？

### C. 代替商品リストのレイアウト
現在: 縦並び（flex flex-col）
```
[画像] 商品A情報           [切り替えボタン]
[画像] 商品B情報           [切り替えボタン]
[画像] 商品C情報           [切り替えボタン]
```

提案1: 横並び（2列グリッド）
```
[画像] 商品A   [ボタン]  |  [画像] 商品B   [ボタン]
```

提案2: 横スクロール
```
[画像] 商品A [ボタン] → [画像] 商品B [ボタン] → [画像] 商品C [ボタン]
```

提案3: 最小限の情報のみ表示
- 表示項目を削減: 商品名 + 価格のみ（コードは省略）
- ボタンをアイコン化

**質問**:
- 代替商品は何件まで表示すべきか？（現在最大3件）
- 横並びにした場合、スマホ表示はどうするか？
- 代替商品の表示項目は何が最低限必要か？（名前、コード、価格、画像）

### D. パディングと余白の調整
- **ボックス内パディング**: p-3 (12px) → p-2 (8px)
- **要素間の間隔**: gap-3 (12px) → gap-2 (8px) または gap-1 (4px)
- **角丸**: rounded-lg → rounded-md（視覚的にコンパクト）

**質問**: 余白をどこまで削減可能か？（視覚的な窮屈さとのバランス）

### E. テキストサイズの調整
現在:
- 見出し: text-sm (14px)
- 商品名: text-sm (14px)
- 詳細情報: text-xs (12px)

提案:
- 見出し: text-xs (12px)
- 商品名: text-xs (12px)
- 詳細情報: text-xs (12px) 維持

**質問**: フォントサイズをどこまで縮小可能か？（可読性の限界）

### F. 通常商品表示のコンパクト化
現在の表示:
```
[画像64x64]  商品名
            コード: XXX
            ¥1,200（税込 ¥1,320）
            [在庫: 100個]
```

提案:
```
[画像48x48]  商品名 | コード: XXX
            ¥1,200（税込 ¥1,320）| 在庫: 100個
```

**質問**: 情報を横並びにして縦サイズを削減してよいか？

## デザイナーに確認していただきたいこと

1. **全体的な方向性**
   - コンパクト化の優先度（情報量 vs スペース削減）
   - モバイル対応はどうするか

2. **具体的なレイアウト案**
   - 廃番商品の情報配置（案A, B, Cなど）
   - 代替商品リストのレイアウト（縦並び or 横並び or スクロール）

3. **デザイントークンの調整**
   - 画像サイズ（48px or 40px）
   - パディング（p-2 or p-1.5）
   - フォントサイズ（text-xs まで縮小可能か）
   - 間隔（gap-2 or gap-1）

4. **優先順位**
   - どの改善が最も効果的か
   - どの情報は削減不可か
